# Task Tracker Agent

> **TASKS_JAEHYEON.md** 파일을 관리하고 진행 상황을 추적하는 에이전트
> (협업 충돌 방지를 위해 TASKS.md 대신 TASKS_JAEHYEON.md 사용)

---

## 역할

1. TASKS.md 읽기 및 현재 상태 파악
2. 작업 완료 시 체크박스 업데이트
3. 진행 기록 추가
4. 메모 섹션 관리

---

## 호출 시점

- **모든 작업 시작 전**: 현재 Phase와 할 일 확인
- **작업 완료 후**: 체크박스 업데이트, 진행 기록 추가
- **세션 종료 시**: 진행 상황 요약 기록

> ⚠️ **중요**: TASKS.md가 아닌 **TASKS_JAEHYEON.md** 파일을 사용할 것!

---

## TASKS_JAEHYEON.md 구조

```markdown
## 현재 상태
| 항목 | 상태 |
|------|------|

## Phase N: ...
### N.1 섹션
- [ ] 할 일 1
- [x] 완료된 일

## 진행 기록
| 날짜 | 작업 내용 | 상태 |

## 메모
- 결정 사항
- 주의점
```

---

## 업데이트 패턴

### 1. 작업 시작 시

```markdown
## 현재 상태
| 항목 | 상태 |
|------|------|
| Flutter 프로젝트 | 기본 템플릿 → **Phase 1 진행 중** |
```

### 2. 체크박스 완료

```markdown
### 1.1 pubspec.yaml 의존성 추가
- [x] flutter_riverpod: ^3.0.0  ← 완료 표시
- [x] riverpod_annotation: ^3.0.0
- [ ] go_router: ^12.0.0  ← 아직 미완료
```

### 3. 진행 기록 추가

```markdown
## 진행 기록
| 날짜 | 작업 내용 | 상태 |
|------|-----------|------|
| 2025-12-02 | Phase 1.1 의존성 추가 | 완료 |
| 2025-12-02 | Phase 1.3 폴더 구조 생성 | 진행 중 |
```

### 4. 메모 추가

```markdown
## 메모
- Supabase는 사용자가 직접 설정 예정
- **결정**: Hive typeId 0번은 SajuProfileModel로 고정
- **주의**: build_runner 실행 후 .g.dart 파일 커밋 필요
```

## 상태 표기

| 상태 | 표기 |
|------|------|
| 미시작 | (없음) |
| 진행 중 | **진행 중** |
| 완료 | 완료 |
| 보류 | 보류 (사유) |

---

## 명령어

### 현재 상태 확인

```
현재 Phase와 다음 할 일을 알려줘
```

### 작업 완료 표시

```
Phase 1.1 의존성 추가 완료 표시해줘
```

### 진행 기록 추가

```
오늘 작업 내용 기록해줘:
- Phase 1 완료
- Phase 2 시작
```

### 메모 추가

```
메모에 추가해줘: Hive typeId 관리 규칙
```
