## 2. 비용 구조: 실제 가격 vs DB 기록 가격

### ⚠️ 핵심 발견: Edge Function 가격 계산 오류

현재 Edge Function 코드에 하드코딩된 가격이 **공식 가격과 다름**.

#### GPT-5.2 (ai-openai-result/index.ts)
| | 코드 적용 가격 | 공식 가격 | 차이 |
|---|---|---|---|
| Input | $3.00/1M | **$1.75/1M** | 코드가 71% 비쌈 |
| Output | $12.00/1M | **$14.00/1M** | 코드가 14% 쌈 |

→ 실제로는 output이 더 비싸므로 **실제 비용이 DB 기록보다 높을 수 있음**

#### Gemini 3.0 Flash (ai-gemini/index.ts)
| | 코드 적용 가격 | 공식 가격 | 차이 |
|---|---|---|---|
| Input | $0.075/1M | **$0.50/1M** | 코드가 **6.7배 저렴** |
| Output | $0.30/1M | **$3.00/1M** | 코드가 **10배 저렴** |

→ **Gemini 비용이 실제로는 DB 기록의 ~7-10배**

## 5. 핵심 문제점

### 문제 1: 수익 < 비용 (적자 구조)
- 사용자당 월 비용: ~$0.97~$2.49 (GPT-5-mini 반영)
- 사용자당 월 광고 수익: ~$0.60~$2.10 (추정)
- **월 적자: 사용자당 ~$0.37~$0.39** (이전 추정보다 개선됨)

### 문제 2: eCPM 불확실성 (한국 시장)
- Native eCPM: $3~7 (비게임 앱 기준)
- Rewarded eCPM: **$13~29** (한국 Android $13.36, iOS $29 - Tier 1급)
- Interstitial eCPM: **$10~15** (한국 Android $11.23)
- 한국은 글로벌 Tier 1~2 시장 (미국, 일본과 비슷한 수준)

### 문제 3: 광고 비율 대비 수익 불균형
- Native 473회 노출 → 14회 클릭 = **CTR 2.96%** (양호)
- Rewarded 19회 시청 → 80,050 토큰 보상 = **토큰 대비 수익 미미**
- Interstitial 26회 → 1회 클릭 = **CTR 3.8%** (양호하나 건수 적음)

### 문제 4: 유료화 수단 부재
- IAP (인앱결제) 미구현 (계획만 있음)
- 구독 모델 미구현
- 프리미엄 티어 미구현

## 6. 비즈니스 모델 개선 전략

### 전략 1: 비용 절감 (즉시 실행 가능)

#### 1-A. Edge Function 가격 수정
```
수정 필요 파일:
- ai-openai/index.ts: $3.00/$12.00 → $1.75/$14.00
- ai-openai-result/index.ts: $3.00/$12.00 → $1.75/$14.00
- ai-gemini/index.ts: $0.075/$0.30 → $0.50/$3.00
```
→ 비용 추적 정확도 개선 (실제 비용 변화 없음, 기록만 정확해짐)

#### 1-B. Gemini 모델 다운그레이드 검토
```
현재: gemini-3-flash-preview ($0.50/$3.00)
대안: gemini-2.5-flash-lite ($0.075/$0.30) ← 10배 저렴
```
- 일운/채팅 품질이 충분하다면 2.5-flash-lite로 전환
- **월 채팅 비용: $0.77 → $0.077 (90% 절감)**

#### 1-C. 채팅 히스토리 압축
- 현재: 전체 히스토리를 매 메시지마다 전송
- 개선: 최근 5개 메시지만 + 요약 컨텍스트
- **Input 토큰 30~50% 절감 가능**

#### 1-D. GPT-5.2 캐시 토큰 활용
- 공식 가격: 캐시 input $0.175/1M (90% 할인)
- 동일 사용자 반복 분석 시 자동 적용
- Responses API background mode에서 캐시 활성화 확인 필요

### 전략 2: 수익 증대 (단기)

#### 2-A. 인앱 결제 (광고 제거) - 최우선
```
상품: "광고 제거" (Non-consumable)
가격: ₩4,900~₩9,900
예상 전환율: 5~10% (22명 중 1~2명)
월 수익: ₩4,900~₩19,800
```
- 이미 계획 문서 존재: `docs/01_Team_TODO/iap_and_i18n_plan.md`
- 패키지: `in_app_purchase` (Google/Apple 공식)

#### 2-B. 구독 모델 도입
```
무료 티어:
- 일 10회 채팅 (50,000 토큰)
- 기본 일운/월운
- 광고 표시

프리미엄 (₩3,900/월 or ₩29,900/년):
- 무제한 채팅
- 광고 제거
- 궁합 무제한
- 카테고리별 상세 운세
- 우선 AI 응답 (GPT-5.2 직접 채팅?)

VIP (₩9,900/월):
- 프리미엄 전부
- 프로필 무제한
- GPT-5.2 직접 상담 (고품질)
- 전용 페르소나
```

#### 2-C. Rewarded Ad 최적화
```
현재: 19회 시청 / 7일 = 2.7회/일
목표: 10회/일 (DAU 22명 기준)
방법:
- 쿼터 80% 도달 시 강제 노출 (현재 선택적)
- 카테고리 운세 열람 시 Rewarded 필수
- 궁합 분석 시 Rewarded 필수
예상 수익 증가: 3~5배
```

### 전략 3: 성장 (중기)

#### 3-A. 일본어/영어 현지화
- 일본 사주 시장 규모: ~3조원 (한국 대비 6배)
- 일본 AdMob eCPM: 한국 대비 2~3배
- **같은 DAU에서 수익 2~3배 가능**

#### 3-B. 바이럴/공유 기능
```
미구현 항목:
- 오늘의 운세 카드 공유 (이미지)
- 궁합 결과 공유 링크
- 추천인 코드 시스템
- SNS 공유 버튼
```

#### 3-C. 푸시 알림 기반 리텐션
```
- "오늘의 운세가 도착했어요" (매일 아침)
- "이번 달 운세가 바뀌었어요" (매월 1일)
- "3일간 방문하지 않았어요" (이탈 방지)
```
